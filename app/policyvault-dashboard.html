How it works

Every rule is enforced on-chain

PolicyVault gives AI agents a safe spending envelope. The program, not the agent, decides what's allowed.

Policy-Controlled Spending

Every consume_budget call is checked against your on-chain policy. Max per request, rolling period cap, and time window are all enforced at the program level — not in your app code.

Max per request0.01 SOL
Period cap (1hr window)0.5 SOL
Vault balance2.0 SOL

Agent Authorization

Only whitelisted agent wallets can call consume_budget. The owner registers each agent as an on-chain AgentApproval PDA.

Replay Protection

Each payment uses a unique nonce, burned on-chain into a NonceRecord PDA. Reusing the same nonce causes an immediate program rejection.

x402 Payment-Gated API Access

The server implements the HTTP 402 flow. Agents pay on-chain and retry with a receipt. The server verifies the tx directly against Solana RPC before releasing data.

1
Agent → GET /data
2
Server ← 402 Payment Required
3
Agent calls consume_budget → tx on Solana
4
Agent → POST /confirm-payment {txSignature}
5
Server verifies on-chain → stores receipt
6
Agent → GET /data?receipt=… → 200

Emergency Pause / Resume

Owner can pause the vault instantly. All consume_budget calls rejected while paused.

On-Chain Receipt Verification

The payment server verifies the tx signature directly via Solana RPC — confirming the tx exists, didn't fail, and called the PolicyVault program.
